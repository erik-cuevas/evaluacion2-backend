Instrucciones Generales
Debes desarrollar una aplicación web utilizando el framework Django.
El sistema representará un módulo de gestión básico de una entidad a tu elección, por ejemplo:
- Gestión de productos en una tienda


La aplicación debe implementar las operaciones de CREAR (Create), LEER (Read) y ELIMINAR (Delete). No se debe implementar la operación UPDATE (Actualizar).
Requisitos de la Aplicación
1. Modelo principal: Definir un modelo con al menos 3 campos (nombre/título, descripción, y un campo extra relevante).
2. Base de datos: Utilizar SQLite y aplicar migraciones.
3. Vistas: (Inicio, Listado, Crear, Eliminar).
4. URLs: Configurar rutas para cada vista.
5. Templates: Usar base.html (navbar), index.html, listado.html y crear.html.
6. Navbar: Con enlaces a Inicio, Listado y Crear.
7. CRUD limitado: Crear registros, leer registros y eliminar registros.

Criterios de Evaluación (25 puntos)
•	Modelo de datos bien definido y coherente (3 puntos).
•	Configuración de URLs y vistas (4 puntos).
•	Formularios funcionando con método POST (3 puntos).
•	Listado correcto de registros desde la base de datos (3 puntos).
•	Función de eliminación implementada (3 puntos).
•	Navbar funcional y navegación clara (2 puntos).
•	Orden, comentarios en el código explicativo (2 puntos).
•	Presentación de su aplicativo al docente (5 puntos)



*nota: tendrá 0,3 décimas adicionales en su evaluación. Si su aplicativo tiene un correcto “look and feel” que tenga relación con su temática. (Ejemplo: si es una pagina de gestión de productos de una tienda que parezca una página web de gestión de productos de una tienda)
